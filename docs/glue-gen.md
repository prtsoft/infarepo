# glue-gen

Generates AWS Glue PySpark scripts and Terraform HCL for every mapping that `pc-extractor` routed to the `GLUE` platform. Each mapping produces one `.py` Glue job script and one `.tf` Terraform resource block.

## Prerequisites

- A `migration-manifest.json` produced by `pc-extractor extract` with scoring applied.
- Mappings must have `target_platform = "GLUE"` to be picked up. Run `pc-extractor ls-mappings --platform GLUE` to see what will be generated.

## Running

The tool is not yet packaged with a `console_scripts` entry point. Run it as a module from the `tools/` directory:

```bash
cd tools/
python -m glue_gen.cli <command> [OPTIONS]
```

## Commands

### `generate-all` — generate all GLUE-routed mappings

```
python -m glue_gen.cli generate-all <manifest_file> [OPTIONS]
```

Iterates every mapping in the manifest that is routed to GLUE and writes a script + Terraform file for each one.

**Options**

| Option | Default | Description |
|---|---|---|
| `--output-dir`, `-o` | `output` | Root directory for all output. |
| `--folder-filter` | (all) | Repeatable. Process only the named folder(s). |
| `--include-review` | false | Also generate stub scripts for REVIEW-flagged mappings. |
| `--verbose`, `-v` | false | Debug logging to stderr. |

**Output layout**

```
<output_dir>/
  glue_jobs/
    <FOLDER_NAME>/
      <MAPPING_NAME>.py
  terraform/
    <FOLDER_NAME>/
      <MAPPING_NAME>.tf
      variables.tf          ← one per folder, declares connection variables
  generation-report.json    ← per-mapping SUCCESS / SKIPPED / ERROR
```

```bash
python -m glue_gen.cli generate-all output/migration-manifest.json \
  --output-dir generated/
```

---

### `generate` — generate a single mapping

```
python -m glue_gen.cli generate <manifest_file> <folder_name> <mapping_name> [OPTIONS]
```

Generates one mapping by name. Useful for iterating on a single problematic mapping without re-running everything.

```bash
python -m glue_gen.cli generate \
  output/migration-manifest.json \
  SALES_MART M_LOAD_FACT_ORDERS \
  --output-dir generated/
```

---

### `preview` — print generated code without writing files

```
python -m glue_gen.cli preview <manifest_file> <folder_name> <mapping_name> [OPTIONS]
```

Prints the generated Glue script (or Terraform HCL with `--terraform`) to stdout. No files are written. Use this to review output before committing to a full generate run.

```bash
# Preview the PySpark script
python -m glue_gen.cli preview \
  output/migration-manifest.json SALES_MART M_LOAD_FACT_ORDERS

# Preview Terraform HCL
python -m glue_gen.cli preview \
  output/migration-manifest.json SALES_MART M_LOAD_FACT_ORDERS --terraform
```

---

### `report` — print summary of a previous generation run

```
python -m glue_gen.cli report <generation_report_json>
```

Pretty-prints the `generation-report.json` from a prior `generate-all` run. Shows SUCCESS / SKIPPED / ERROR per mapping.

```bash
python -m glue_gen.cli report generated/generation-report.json
```

## Generated Glue script structure

Each `.py` output follows this layout:

```python
# ============================================================
# Glue job: M_LOAD_FACT_ORDERS  (SALES_MART)
# Complexity score: 3  |  Platform: GLUE
# Generated by glue-gen  |  Review warnings below
# ============================================================

import sys
from awsglue.transforms import *
from awsglue.utils import getResolvedOptions
from pyspark.context import SparkContext
from awsglue.context import GlueContext
from awsglue.job import Job
import pyspark.sql.functions as F

# --- Job args (set these in the Glue job definition or Terraform) ---
args = getResolvedOptions(sys.argv, [
    "JOB_NAME",
    "SALESDB_CONNECTION",
    "MARTDB_CONNECTION",
    "START_DATE",          # ← from $$START_DATE mapping parameter
])

sc = SparkContext()
glueContext = GlueContext(sc)
spark = glueContext.spark_session
job = Job(glueContext)
job.init(args["JOB_NAME"], args)

# --- Source read ---
df_src_orders = glueContext.create_dynamic_frame.from_options(
    connection_type="jdbc",
    connection_options={
        "url": args["SALESDB_CONNECTION"],
        "dbtable": "dbo.ORDERS",
    },
).toDF()

# --- Transformations ---
df = df_src_orders
df = df.filter(F.col("STATUS") != "CANCELLED")
df = df.withColumn("LOAD_TS", F.current_timestamp())

# --- Target write ---
df.write.format("jdbc").option("url", args["MARTDB_CONNECTION"]) \
    .option("dbtable", "dbo.TGT_FACT_ORDERS").mode("overwrite").save()

job.commit()
```

### Warnings in generated scripts

When the generator cannot fully translate a construct, it inserts a `# TODO:` comment in the script and records the issue in `generation-report.json`. Common warning causes:

- Unresolved expression functions (unknown PC built-ins)
- Stored Procedure transformations in a GLUE-routed mapping (should be REVIEW)
- Multi-source Joiners where join key resolution is ambiguous
- Normalizer or XML transformations

Always review warnings before running a generated script.

## Terraform output

Each `<MAPPING_NAME>.tf` file defines one `aws_glue_job` resource:

```hcl
resource "aws_glue_job" "sales_mart_m_load_fact_orders" {
  name     = "SALES_MART-M_LOAD_FACT_ORDERS"
  role_arn = var.glue_role_arn

  command {
    script_location = "s3://${var.glue_scripts_bucket}/SALES_MART/M_LOAD_FACT_ORDERS.py"
    python_version  = "3"
  }

  default_arguments = {
    "--job-bookmark-option" = "job-bookmark-enable"
    "--SALESDB_CONNECTION"  = var.SALESDB_CONNECTION
    "--MARTDB_CONNECTION"   = var.MARTDB_CONNECTION
  }
}
```

The `variables.tf` in each folder directory declares the connection variables referenced across that folder's jobs. You wire these to your actual connection strings in a `terraform.tfvars` or via AWS Secrets Manager / Parameter Store (see [TODOS.md](../TODOS.md)).

## Python API

```python
from glue_gen.generator import generate_all, generate_single
from glue_gen.code_builder import build_glue_script
from glue_gen.tf_builder import build_terraform_job
import json
from pathlib import Path

with open("output/migration-manifest.json") as f:
    data = json.load(f)

# Generate everything
from glue_gen.cli import _load_manifest
manifest = _load_manifest(data)
report = generate_all(manifest, output_dir=Path("generated/"))

# Or inspect one script without writing
folder = manifest.folders["SALES_MART"]
mapping = folder.mappings["M_LOAD_FACT_ORDERS"]
script_text, warnings = build_glue_script(mapping, folder)
print(script_text)
```
