# databricks-gen

Generates Databricks Python notebooks and Terraform HCL for every mapping that `pc-extractor` routed to the `DATABRICKS` platform. Primarily used for lakehouse targets — Delta Lake on S3, Parquet/ORC via Glue Catalog, and similar.

## Prerequisites

- A `migration-manifest.json` from `pc-extractor extract` with scoring applied.
- Mappings must have `target_platform = "DATABRICKS"`. The scorer sets this when targets are Delta/S3/Parquet/ORC/Iceberg/Hive, or when the mapping name contains lakehouse hints (`extract`, `delta`, `lake`, etc.).
- Check what will be generated: `pc-extractor ls-mappings --platform DATABRICKS`

## Running

```bash
cd tools/
python -m databricks_gen.cli <command> [OPTIONS]
```

## Commands

### `generate-all`

```
python -m databricks_gen.cli generate-all <manifest_json> [OPTIONS]
```

Generates notebooks and Terraform for all DATABRICKS-routed mappings.

**Options**

| Option | Default | Description |
|---|---|---|
| `--output-dir`, `-o` | `output` | Root directory for all output. |
| `--folder-filter` | (all) | Repeatable. Process only the named folder(s). |
| `--verbose`, `-v` | false | Debug logging to stderr. |

**Output layout**

```
<output_dir>/
  notebooks/
    <FOLDER_NAME>/
      <MAPPING_NAME>.py         ← Databricks Python notebook
  terraform/
    <FOLDER_NAME>/
      <MAPPING_NAME>.tf
      variables.tf
  databricks-generation-report.json
```

```bash
python -m databricks_gen.cli generate-all output/migration-manifest.json \
  --output-dir generated/ \
  --folder-filter PATIENT_EXTRACT
```

---

### `generate` — generate a single mapping

```
python -m databricks_gen.cli generate <manifest_json> <folder> <mapping> [OPTIONS]
```

**Options**

| Option | Description |
|---|---|
| `--output-dir`, `-o` | Output directory. |
| `--preview` | Print the notebook to stdout instead of writing files. |

```bash
# Write files
python -m databricks_gen.cli generate \
  output/migration-manifest.json \
  PATIENT_EXTRACT M_EXTRACT_PATIENT_DELTA

# Preview without writing
python -m databricks_gen.cli generate \
  output/migration-manifest.json \
  PATIENT_EXTRACT M_EXTRACT_PATIENT_DELTA --preview
```

## Generated notebook structure

Each output notebook is a Python file importable into Databricks as a `.py` notebook (Databricks Source Format). It follows this layout:

```python
# Databricks notebook source
# MAGIC %md
# ## M_EXTRACT_PATIENT_DELTA (PATIENT_EXTRACT)
# **Complexity score:** 4  |  **Platform:** DATABRICKS
# **⚠ HIPAA:** This notebook processes PHI data.
#    Do not log field values. Apply column-level encryption before writing.
# Generated by databricks-gen

# COMMAND ----------
# Imports
from pyspark.sql import SparkSession
import pyspark.sql.functions as F
from delta.tables import DeltaTable

# COMMAND ----------
# Parameters (set via Databricks Widgets or job parameters)
dbutils.widgets.text("load_date",   "", "Load Date ($$LOAD_DATE)")
dbutils.widgets.text("source_dsn",  "", "Source connection string")
load_date  = dbutils.widgets.get("load_date")
source_dsn = dbutils.widgets.get("source_dsn")

# COMMAND ----------
# Source read: SRC_PATIENTS (Microsoft SQL Server)
df_src_patients = (
    spark.read.format("jdbc")
    .option("url", source_dsn)
    .option("dbtable", "dbo.PATIENTS")
    .load()
)

# COMMAND ----------
# Transformations
df = df_src_patients
df = df.withColumn("LOAD_TS", F.current_timestamp())

# COMMAND ----------
# Target write: TGT_PATIENT_DELTA (Delta)
(
    df.write
    .format("delta")
    .mode("append")
    .option("mergeSchema", "true")
    .save("/mnt/delta/patient_extract/patient_delta")
)

# COMMAND ----------
# Summary
print(f"Rows written: {df.count():,}")
```

### HIPAA handling

When the mapping's target fields contain PHI-like column names (`PATIENT_ID`, `FULL_NAME`, `DOB`, `MRN`, `SSN`, etc.), the generator:

1. Adds a `%md` HIPAA warning cell at the top of the notebook.
2. Suppresses display output that would show row values.
3. Adds a comment block recommending column-level encryption before writing to Delta.

This is a best-effort annotation — it does not substitute for a formal data classification review.

## Terraform output

Each `<MAPPING_NAME>.tf` defines one `databricks_job` resource:

```hcl
resource "databricks_job" "patient_extract_m_extract_patient_delta" {
  name = "PATIENT_EXTRACT-M_EXTRACT_PATIENT_DELTA"

  task {
    task_key = "main"
    notebook_task {
      notebook_path = "/Repos/migrations/PATIENT_EXTRACT/M_EXTRACT_PATIENT_DELTA"
    }
    existing_cluster_id = var.databricks_cluster_id
  }

  schedule {
    quartz_cron_expression = var.patient_extract_schedule
    timezone_id            = "America/New_York"
  }
}
```

## Python API

```python
from databricks_gen.notebook_builder import DatabricksNotebookBuilder, render_notebook
from databricks_gen.generator import generate_all
import json
from pathlib import Path

with open("output/migration-manifest.json") as f:
    data = json.load(f)

from databricks_gen.cli import _load_manifest
manifest = _load_manifest(data)

# Preview one notebook
folder = manifest.folders["PATIENT_EXTRACT"]
mapping = folder.mappings["M_EXTRACT_PATIENT_DELTA"]

builder = DatabricksNotebookBuilder()
notebook = builder.build(mapping)

print(render_notebook(notebook))
for w in notebook.warnings:
    print(f"  WARNING: {w}")

# Generate everything
report = generate_all(manifest, output_dir=Path("generated/"))
print(f"Generated: {report.generated}  Skipped: {report.skipped}  Errors: {report.errors}")
```
